# Лабораторная №5 - Управление коллекцией маршрутов

## О проекте
Консольное приложение для управления коллекцией объектов `Route` (маршруты). Программа позволяет добавлять, удалять, изменять и просматривать маршруты, а также сохранять их в CSV-файл и загружать из него.

## Функциональность
- **Интерактивный режим** - работа с пользователем через консоль
- **Режим выполнения скриптов** - чтение команд из файла
- **Цветной вывод** - сообщения разных цветов
- **Сохранение/загрузка** в CSV-формате
- **Автоматическая генерация ID**
- **Автодополнение команд** - терминал автоматически дополняет команду до целого слова по `Tab`
- **Кэширование через Redis** - курс доллара кэшируется на 10 минут, повторные запросы не делают HTTP-вызов
- **Fallback без Redis** - если Redis недоступен, приложение продолжает работу через прямой HTTP-запрос

## Команды

| Команда | Описание |
|---------|----------|
| `add` | Добавляет новый элемент |
| `add_if_max {element}` | Добавляет элемент, если он максимальный |
| `average_of_distance` | Среднее значение distance |
| `clear` | Очищает коллекцию |
| `count {distance}` | Выводит количество элементов с полем distance равным данному |
| `execute_script file_name` | Выполняет скрипт из файла |
| `exit` | Завершает работу |
| `filter_less_than_distance distance` | Выводит элементы с distance < заданного |
| `help` | Справка по командам |
| `history` | Последние 14 команд |
| `info` | Информация о коллекции |
| `remove_all_by_distance distance` | Удаляет элементы с указанным distance |
| `remove_by_id id` | Удаляет элемент по ID |
| `remove_first` | Удаляет первый элемент |
| `save [file_name]` | Сохраняет коллекцию в файл |
| `show` | Выводит все элементы с курсом в рублях |
| `test1` | Выводит все элементы без Redis (прямой HTTP-запрос на каждый элемент) |
| `test2` | Выводит все элементы с Redis-кэшем + замер времени выполнения |
| `update id {element}` | Обновляет элемент по ID |

## Redis

Приложение использует Redis для кэширования курса доллара ЦБ РФ.

**Установка и запуск:**
```bash
sudo apt install redis-server
sudo systemctl start redis-server
```

**Остановить:**
```bash
sudo systemctl stop redis-server
```

**Автозапуск при старте системы:**
```bash
sudo systemctl enable redis-server
```

Если Redis не запущен — приложение автоматически делает прямой HTTP-запрос и продолжает работу в штатном режиме.

## Запуск

Задать переменную окружения с путём к CSV-файлу:
```bash
export JAVAMETASPACE=/path/to/file.csv
```
Затем запустить jar:
```bash
java -jar Lab5.jar
```

Название переменной окружения можно изменить в `Runner.java`.
